# DAMNA – Blood Donation Android Application

DAMNA is an Android mobile application designed to simplify the blood donation process by connecting donors and recipients through a centralized and user-friendly platform.

The application allows users to request blood donations, browse donation opportunities, track request status, and manage donor-related information through an organized mobile interface.

## Overview

DAMNA was developed as a native Android application focused on improving accessibility and communication during blood donation situations. The system aims to provide a faster and more structured way for individuals to connect during urgent blood requests.

The platform includes donor and recipient workflows, blood request management, eligibility checking, request tracking, and user account functionality.

## Core Features

* User authentication and login
* Blood donation request system
* Blood request tracking
* Donor eligibility checking
* Search and filtering functionality
* Dashboard for navigating application features
* Donor and recipient profile management
* Request status monitoring
* Fingerprint authentication support
* Settings and account management

## Technology Stack

Frontend & Mobile Development:

* Android Studio
* Java
* XML Layouts

Backend & Services:

* Firebase Authentication
* Firebase Realtime Database / Firestore

## Application Structure

The application is organized into multiple Android activities, including:

* DashboardActivity – main application dashboard
* LoginActivity – user authentication
* DonateBloodActivity – donor workflow
* RequestBloodActivity – blood request submission
* RequestStatusActivity – request tracking
* EligibilityActivity – donor eligibility checking
* SearchActivity – search functionality
* FingerprintActivity – biometric authentication
* SettingsActivity – user settings management

The application also includes reusable models and adapters for managing blood requests, donor data, and search functionality.

## Security

The system includes:

* User authentication
* Biometric authentication support
* User-specific data handling
* Validation for donation and request workflows
